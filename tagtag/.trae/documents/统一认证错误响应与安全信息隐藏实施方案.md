## 目标
- 登录与刷新场景对“用户不存在”和“密码错误”等认证失败统一返回信息：“凭证无效”，避免泄漏账号存在性。
- 保持 HTTP 状态与全局处理一致（统一 401），前后端协议稳定。

## 关键设计说明
- 当前 `ErrorCode` 的 `code` 即 HTTP 状态码；新增业务枚举如 `USER_NOT_FOUND`、`INVALID_CREDENTIALS` 会与 401 冲突并破坏 `fromCode` 索引。
- 采取“统一使用 `ErrorCode.UNAUTHORIZED`，通过 `BusinessException(ErrorCode.UNAUTHORIZED, "凭证无效")` 传递统一消息”的方式实现安全隐藏，无需改动枚举结构。

## 修改点
1. 服务层统一异常信息
   - 文件：`tagtag-module/tagtag-module-auth/src/main/java/dev/tagtag/module/auth/service/impl/AuthServiceImpl.java`
   - 改动：
     - 用户名/密码为空 → `BusinessException(ErrorCode.BAD_REQUEST, "用户名或密码不能为空")`
     - 用户不存在/凭证缺失/密码错误 → `BusinessException(ErrorCode.UNAUTHORIZED, "凭证无效")`
     - 刷新令牌无效 → `BusinessException(ErrorCode.UNAUTHORIZED, "凭证无效")`
   - 方法均保留/完善函数级注释。

2. 全局异常处理无需变更
   - `GlobalExceptionHandler` 已按 `BusinessException` 返回 `{code, message}` 且状态码为 `ex.getHttpStatus()`，保持不变即可。

3. 测试用例
   - 新增/完善登录接口测试：
     - 用户名为空/密码为空 → 400 + "用户名或密码不能为空"
     - 用户不存在 → 401 + "凭证无效"
     - 密码错误 → 401 + "凭证无效"
     - 刷新令牌无效 → 401 + "凭证无效"
