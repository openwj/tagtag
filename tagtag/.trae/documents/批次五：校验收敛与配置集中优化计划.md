## 目标
- 收敛参数校验到 DTO 与控制器，服务层仅保留防御性校验。
- 集中分页默认值与排序白名单到配置类，统一注入使用。
- 统一常量与文案，消除魔法字符串；持续清理未使用 XML 简单查询。

## 调整项
1. DTO 与控制器校验
- 为 Dept/Role/Menu 等请求 DTO 增补 `jakarta.validation` 注解（`@NotBlank/@Size/@NotNull/@Positive`等）。
- 控制器方法参数统一 `@Valid`，类级启用 `@Validated`；服务层仅保留空/ID 防御性校验。
- 函数级注释：说明每个校验字段的业务含义与约束。

2. 分页与排序配置集中
- 新增 `PageProperties`（`framework/config`）：包含 `defaultPageSize`、`maxPageSize`、各模块排序白名单；使用 `@ConfigurationProperties(prefix="page")` 注入。
- 服务层调用处将现有 `filterSortByWhitelist(...)` 替换为读取 `PageProperties` 的白名单；分页默认值通过属性注入替代硬编码。

3. 常量与文案统一
- 控制器回包统一使用 `AppMessages`（创建/更新/删除/分配成功），扩展到 Dept/System 控制器。
- Claims 读写剩余位置统一使用 `SecurityClaims`，消除字符串字面量。

4. 简单查询与 XML 清理
- 巡检 Role/Menu/Dept 是否尚有“按 code/name/parentId/menuCode”的简单 XML 查询与 Mapper 方法：若未被引用，删除 XML 节点与方法声明；用 LambdaQuery 完成简单查询。
- 保留分页与联表/批量 XML，不做改动。

5. 缓存策略微调（可选）
- 已有 `roleMenuCodes/roleMenus/deptTree`：评估菜单树或高频只读接口是否需要只读缓存；在变更处统一 `@CacheEvict`。
