Tagtag 框架全面支持 AI 辅助开发，通过集成 Model Context Protocol (MCP) 和提供 AI 友好的文档，让您能够充分利用 AI 工具来加速开发流程。

## 什么是 MCP？

Model Context Protocol (MCP) 是一个开放标准，用于将 AI 助手（如 Claude、ChatGPT 等）连接到外部数据源和工具。通过 MCP，AI 可以直接访问您的项目文档、代码库和其他资源，从而提供更准确、更相关的帮助。

## Tagtag 的 AI 支持特性

### 1. AI 友好的文档

Tagtag 官方文档已经集成了 `nuxt-llms` 模块，自动生成 AI 可读的文档文件：

- **llms.txt**: 简洁的结构化文档，适合 AI 快速理解项目
- **llms-full.txt**: 完整的详细文档，提供深度的项目信息

访问地址：
- http://tagtag.cn/llms.txt
- http://tagtag.cn/llms-full.txt

这些文档遵循 llms.txt 标准，可以被各种 AI 工具和 MCP 服务器直接读取和解析。

### 2. MCP 工具集成

Tagtag 文档站点已经配置了 `@nuxtjs/mcp-toolkit`，提供了标准的 MCP 工具接口，允许 AI 助手直接访问和查询文档内容。

## 如何在 Trae 中使用 AI 辅助开发

### 步骤 1: 配置 MCP 服务器

在 Trae 中，您需要配置 MCP 服务器来连接到 Tagtag 的文档资源。

#### MCP 配置示例

创建或编辑您的 MCP 配置文件（通常在用户配置目录）：

```json
{
  "mcpServers": {
    "tagtag-docs": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "http://tagtag.cn/llms.txt"
      ],
      "env": {}
    }
  }
}
```

### 步骤 2: 在 Trae 中启用 MCP

1. 打开 Trae 的设置
2. 导航到 "AI" 或 "MCP" 配置部分
3. 添加 Tagtag 文档的 MCP 服务器配置
4. 保存并重启 Trae

### 步骤 3: 使用 AI 助手

配置完成后，您就可以在 Trae 中使用 AI 助手了：

```typescript
// 示例：询问 AI 如何创建新的 CRUD 模块
"请帮我创建一个用户管理模块，包含增删改查功能"
```

AI 会自动访问 Tagtag 文档，理解框架结构，并为您提供准确的代码示例和指导。

## 最佳实践

当向 AI 提问时，提供足够的上下文信息：


### 集成到 CI/CD

将 AI 辅助开发集成到您的 CI/CD 流程中：

```yaml
# .github/workflows/ai-review.yml
name: AI Code Review
on: [pull_request]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: AI Review
        run: |
          npx @modelcontextprotocol/cli review \
            --docs http://tagtag.cn/llms.txt
```

## 常见问题

### Q: AI 生成的代码可以直接使用吗？

A: 不建议直接使用。AI 生成的代码需要经过人工审查、测试和优化，确保符合项目规范和业务需求。

### Q: 如何提高 AI 的回答质量？

A: 提供清晰、具体的上下文信息，参考 Tagtag 的文档结构，使用框架术语，这样 AI 能提供更准确的回答。

### Q: MCP 服务器连接失败怎么办？

A: 检查网络连接，确认文档地址可访问，查看 MCP 服务器日志，确保配置正确。

### Q: 可以离线使用 AI 辅助开发吗？

A: 可以。您可以下载 llms.txt 文件到本地，配置本地 MCP 服务器，实现离线访问。

## 总结

Tagtag 框架通过以下方式支持 AI 辅助开发：

1. **AI 友好的文档**：提供标准的 llms.txt 和 llms-full.txt 文档
2. **MCP 集成**：支持 Model Context Protocol，方便 AI 工具访问
3. **清晰的代码结构**：便于 AI 理解和生成代码
4. **完善的文档**：覆盖所有核心功能和最佳实践

通过合理使用 AI 辅助开发，您可以：

- 提高开发效率
- 减少重复工作
- 学习最佳实践
- 加速项目交付
