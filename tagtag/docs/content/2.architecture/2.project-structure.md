# 项目结构

Tagtag 项目包含后端和前端两个主要部分，以及完善的文档系统。

## 目录概览

```text
tagtag
├── docs                      # 文档项目 (Nuxt Content)
├── tagtag-ui                 # 前端项目 (Vue 3 + Vite, Monorepo)
│   ├── apps                  # 应用入口
│   ├── packages              # 核心包与 UI 组件库
│   └── internal              # 构建与工具脚本
├── tagtag-common             # 后端：通用工具层（纯 Java 工具，无 Spring 依赖）
├── tagtag-framework          # 后端：基础设施层（Spring Boot, Security, MyBatis Plus）
├── tagtag-kernel             # 后端：核心业务支持（注解, 枚举, 常量, 上下文）
├── tagtag-contract           # 后端：API 定义与契约
│   ├── tagtag-contract-auth  # 认证契约
│   ├── tagtag-contract-iam   # 身份与访问管理契约
│   ├── tagtag-contract-storage # 存储契约
│   └── tagtag-contract-system # 系统契约
├── tagtag-module             # 后端：业务实现
│   ├── tagtag-module-auth    # 认证模块实现
│   ├── tagtag-module-iam     # 身份与访问管理模块实现
│   ├── tagtag-module-storage # 存储模块实现
│   └── tagtag-module-system  # 系统模块实现
├── tagtag-start              # 后端：应用启动入口与配置
└── pom.xml                   # 后端：根 Maven POM 文件
```

## 后端模块依赖关系

- `tagtag-start` 依赖所有 `tagtag-module-*`。
- `tagtag-module-*` 依赖 `tagtag-contract-*`、`tagtag-framework` 和 `tagtag-kernel`。
- `tagtag-contract-*` 依赖 `tagtag-common`。
- `tagtag-framework` 依赖 `tagtag-common`。
- `tagtag-kernel` 依赖 `tagtag-common`。

## 后端模块详解

### 1. 基础设施层 (`tagtag-framework`)

提供整个应用的基础技术支持，主要包含：
- **Web**: 全局异常处理、CORS 配置、TraceId 过滤器。
- **Security**: Spring Security 配置、JWT 认证服务、权限校验 (Guard)、用户上下文 (UserPrincipal)。
- **Config**: Redis、MyBatis Plus、Jackson、Cache 等配置类。
- **AOP**: 限流切面 (RateLimitAspect) 等。

### 2. 核心支持层 (`tagtag-kernel`)

提供业务开发所需的通用支持，主要包含：
- **Annotation**: 自定义注解 (如 `@RateLimit`)。
- **Constant**: 系统常量 (错误码、权限码、角色码)。
- **Enums**: 枚举定义 (如日志类型)。
- **Util**: 用户上下文持有者 (UserContextHolder) 等业务工具。

### 3. 通用工具层 (`tagtag-common`)

纯 Java 工具类库，不依赖 Spring 框架，提供最基础的工具支持。

## 前端模块详解

`tagtag-ui` 采用 Monorepo 结构管理，主要目录说明如下：

### 1. 应用入口 (`apps`)

- **`apps/tagtag`**: 主应用程序入口。
  - `src/api`: API 请求定义。
  - `src/views`: 页面视图组件。
  - `src/router`: 路由配置。
  - `src/store`: Pinia 状态管理。

### 2. 核心包 (`packages`)

- **`packages/@core`**: 核心共享库。
  - **`base`**: 基础设计系统、图标和共享工具。
  - **`ui-kit`**: 封装的 UI 组件库。
    - `shadcn-ui`: 基于 Shadcn UI 的基础组件。
    - `form-ui`: 表单组件封装。
    - `layout-ui`: 布局组件封装。
    - `popup-ui`: 弹窗、抽屉等组件封装。
  - **`effects`**: 业务副作用与逻辑封装。
    - `access`: 权限控制逻辑。
    - `hooks`: 通用 Hooks (如 `useTabs`, `useRefresh`)。
    - `request`: 请求库封装。
    - `plugins`: 第三方插件集成 (Echarts, Motion, Vxe-table)。
  - **`preferences`**: 应用偏好设置管理。

### 3. 内部工具 (`internal`)

- **`internal/lint-configs`**: ESLint, Prettier, Stylelint 等代码规范配置。
- **`internal/vite-config`**: Vite 构建配置封装。
- **`internal/tailwind-config`**: Tailwind CSS 配置封装。
