# Quick Start

This guide will help you set up the Tagtag backend services and frontend application on your local machine.

## Prerequisites

Ensure you have the following installed:

### Backend
- **Java JDK 17+**
- **Maven 3.8+**
- **MySQL 8.0+**
- **Redis 6.0+**

### Frontend
- **Node.js >= 20.12.0**
- **pnpm >= 10.0.0**

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/admin/openwj/tagtag.git
cd tagtag
```

### 2. Initialize Database (Backend)

1.  **Create Database**: Create a MySQL database named `tagtag`.
    ```sql
    CREATE DATABASE tagtag CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```

2.  **Automatic Initialization**: Tagtag uses Spring Boot's automatic initialization feature, which executes database scripts automatically when the application starts.

    **Script Locations**:
    - Table structure scripts: `src/main/resources/db/schema.sql` under each module
    - Initial data scripts: `src/main/resources/db/data/` directory under each module

    **Auto-loading Configuration** (already configured in `application-dev.yml`):
    ```yaml
    spring:
      datasource:
        init:
          mode: always
          schema-locations:
            - classpath*:db/schema.sql
          data-locations:
            - classpath*:db/data/**.sql
    ```

3.  **Initial Data Description**:
    - Admin account: `admin` / `admin123`
    - Normal user account: `user` / `user123`
    - Initial roles: Super Admin, Admin, Normal User
    - Initial menus: System Management, User Management, Role Management, Menu Management, etc.

### 3. Configure Application (Backend)

Edit `tagtag-start/src/main/resources/application.yml` to match your local environment (Database credentials, Redis host).

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/tagtag?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
  data:
    redis:
      host: localhost
      port: 6379
```

### 4. Build and Run Backend

```bash
mvn clean package -DskipTests
java -jar tagtag-start/target/tagtag-start-0.0.1-SNAPSHOT.jar
```

Alternatively, run `TagtagApplication.java` directly from your IDE.

### 5. Start Frontend Application

1.  Enter the frontend directory:
    ```bash
    cd tagtag-ui
    ```

2.  Install dependencies:
    ```bash
    pnpm install
    ```

3.  Start the development server:
    ```bash
    pnpm run dev
    ```

## Verification

Once started:

- **Frontend Application**: Access [http://localhost:5666](http://localhost:5666)
- **Backend API Docs**: Access [http://localhost:8080/doc.html](http://localhost:8080/doc.html) (Swagger/Knife4j)
- **Health Check**: Access [http://localhost:8080/actuator/health](http://localhost:8080/actuator/health)

## Common Issues and Solutions

### 1. Database Connection Failure

**Problem Description**: Database connection error when starting the application.

**Solutions**:
- Check if MySQL service is running normally
- Verify database name, username, and password are correct
- Check if database address and port are correct
- Confirm the database user has sufficient permissions

### 2. Redis Connection Failure

**Problem Description**: Redis connection error when starting the application.

**Solutions**:
- Check if Redis service is running normally
- Verify Redis address and port are correct
- Check Redis password (if set)
- Confirm Redis instance allows remote connections

### 3. Frontend Cannot Access Backend API

**Problem Description**: Frontend application cannot connect to backend API.

**Solutions**:
- Check if backend service is running normally
- Verify the API address configured in the frontend is correct
- Check CORS configuration
- Check firewall settings to ensure ports are not blocked

### 4. Port Conflict When Starting

**Problem Description**: Port already in use error when starting the application.

**Solutions**:
- Modify backend port: Edit `server.port` configuration in `application.yml`
- Modify frontend port: Edit `PORT` configuration in `tagtag-ui/apps/tagtag/.env`
- Find and terminate the process occupying the port

### 5. Maven Dependency Download Failure

**Problem Description**: Dependency download failure when executing `mvn clean package`.

**Solutions**:
- Check network connection
- Clean Maven local repository: Delete `~/.m2/repository` directory and re-download dependencies
- Configure Maven mirror: Configure domestic mirror source in `settings.xml`

### 6. pnpm Dependency Installation Failure

**Problem Description**: Dependency installation failure when executing `pnpm install`.

**Solutions**:
- Check network connection
- Clean pnpm cache: Execute `pnpm store prune`
- Upgrade pnpm version: Execute `npm install -g pnpm`
- Configure pnpm mirror: Configure domestic mirror source in `.npmrc`

### 7. Compilation Error

**Problem Description**: Error during compilation.

**Solutions**:
- Check if JDK version meets requirements (JDK 17+)
- Check if Node.js version meets requirements (Node.js >= 20.12.0)
- Clean compilation cache: Execute `mvn clean` or `pnpm run clean`
- Check for syntax errors or type errors in code

### 8. Login Failure

**Problem Description**: Login failure using initial account.

**Solutions**:
- Confirm account and password are correct (initial account: `admin` / `admin123`)
- Check if backend service is running normally
- Check if user data exists in database
- Check JWT configuration
