# Project Structure

The Tagtag project consists of two main parts: Backend and Frontend, along with a comprehensive documentation system.

## Directory Overview

```text
tagtag
├── docs                      # Documentation Project (Nuxt Content)
├── tagtag-ui                 # Frontend Project (Vue 3 + Vite, Monorepo)
│   ├── apps                  # Application Entries
│   ├── packages              # Core Packages & UI Components
│   └── internal              # Build & Tool Scripts
├── tagtag-common             # Backend: Utilities (Pure Java, No Spring)
├── tagtag-framework          # Backend: Infrastructure (Spring Boot, Security, MyBatis Plus)
├── tagtag-kernel             # Backend: Core Business Support (Annotations, Enums, Constants, Context)
├── tagtag-contract           # Backend: API Definitions & Contracts
│   ├── tagtag-contract-auth  # Authentication Contract
│   ├── tagtag-contract-iam   # Identity & Access Management Contract
│   ├── tagtag-contract-storage # Storage Contract
│   └── tagtag-contract-system # System Contract
├── tagtag-module             # Backend: Business Implementation
│   ├── tagtag-module-auth    # Authentication Module Implementation
│   ├── tagtag-module-iam     # IAM Module Implementation
│   ├── tagtag-module-storage # Storage Module Implementation
│   └── tagtag-module-system  # System Module Implementation
├── tagtag-start              # Backend: Application Entry Point & Config
└── pom.xml                   # Backend: Root Maven POM
```

## Backend Module Dependencies

- `tagtag-start` depends on all `tagtag-module-*`.
- `tagtag-module-*` depends on `tagtag-contract-*`, `tagtag-framework`, and `tagtag-kernel`.
- `tagtag-contract-*` depends on `tagtag-common`.
- `tagtag-framework` depends on `tagtag-common`.
- `tagtag-kernel` depends on `tagtag-common`.

## Backend Module Details

### 1. Infrastructure Layer (`tagtag-framework`)

Provides foundational technical support for the entire application, primarily containing:
- **Web**: Global exception handling, CORS config, TraceId filter.
- **Security**: Spring Security config, JWT authentication service, Permission validation (Guard), User Context (UserPrincipal).
- **Config**: Configuration classes for Redis, MyBatis Plus, Jackson, Cache, etc.
- **AOP**: Aspects like Rate Limit (RateLimitAspect).

### 2. Core Support Layer (`tagtag-kernel`)

Provides common support required for business development, primarily containing:
- **Annotation**: Custom annotations (e.g., `@RateLimit`).
- **Constant**: System constants (Error codes, Permission codes, Role codes).
- **Enums**: Enumeration definitions (e.g., Log types).
- **Util**: Business tools like User Context Holder (UserContextHolder).

### 3. Utility Layer (`tagtag-common`)

A pure Java utility library that does not depend on the Spring framework, providing the most basic utility support.

## Frontend Module Details

`tagtag-ui` uses a Monorepo structure, with key directories described as follows:

### 1. Applications (`apps`)

- **`apps/tagtag`**: The main application entry.
  - `src/api`: API request definitions.
  - `src/views`: Page view components.
  - `src/router`: Routing configuration.
  - `src/store`: Pinia state management.

### 2. Core Packages (`packages`)

- **`packages/@core`**: Core shared libraries.
  - **`base`**: Basic design system, icons, and shared utilities.
  - **`ui-kit`**: Encapsulated UI component library.
    - `shadcn-ui`: Base components based on Shadcn UI.
    - `form-ui`: Form component encapsulation.
    - `layout-ui`: Layout component encapsulation.
    - `popup-ui`: Popup, Drawer, etc. component encapsulation.
  - **`effects`**: Business side-effects and logic encapsulation.
    - `access`: Permission control logic.
    - `hooks`: Common Hooks (e.g., `useTabs`, `useRefresh`).
    - `request`: Request library encapsulation.
    - `plugins`: Third-party plugin integration (Echarts, Motion, Vxe-table).
  - **`preferences`**: Application preference management.

### 3. Internal Tools (`internal`)

- **`internal/lint-configs`**: Configuration for ESLint, Prettier, Stylelint, etc.
- **`internal/vite-config`**: Vite build configuration encapsulation.
- **`internal/tailwind-config`**: Tailwind CSS configuration encapsulation.
